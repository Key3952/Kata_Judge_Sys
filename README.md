# ⚖️ Система Судейства Дзюдо Ката

Локальная система судейства для соревнований по Дзюдо Ката, работающая без интернета по локальной сети.

## 🎯 Основные Возможности

✅ **Административная часть:**
- Создание и управление соревнованиями
- Регистрация участников и судей с автозаполнением
- Управление дисциплинами из словаря technics.py
- Закрытие соревнований

✅ **Судейская часть:**
- Интерактивная форма оценки с кликабельными штрафами
- Реальное обновление оценок
- Сохранение данных в CSV файлы

✅ **Публичное табло:**
- Итоговая таблица результатов
- Автоматический расчет баллов (среднее 3-х оценок)
- Сортировка по местам
- Автоматическое обновление каждые 10 секунд

## 📋 Рабочий Процесс

### Этап 1: Создание Соревнования (Администратор)

1. Перейди в `/dashboard/admin`
2. Нажми "Создать Соревнование"
3. Заполни: Название, Путь сохранения, Баннер
4. Нажми "Создать"

### Этап 2: Добавление Дисциплин

1. В редакторе соревнования нажми "Добавить Дисциплину"
2. Выбери из списка (Nage-no-kata, Katame-no-kata и т.д.)
3. Система создаст необходимые папки и файлы

### Этап 3: Регистрация Участников

1. Нажми "Регистрация" для дисциплины
2. Добавь пары (Тори и Уке)
3. Начни вводить ФИО - система подскажет из глобального CSV
4. При полном совпадении заполнятся все поля автоматически
5. Добавь 5 судей аналогично
6. Нажми "Сохранить"

### Этап 4: Судейство

1. Перейди на `/dashboard` (публичная панель)
2. Выбери соревнование и дисциплину
3. Нажми "Форма судьи"
4. Выбери свою пару и место (1-5)
5. Оценивай техники:
   - Нажимай кнопки штрафов (-0.5, -1, -3, -5)
   - Отмечай "Забытая техника" если нужно (= 0 + итог ÷2)
   - Видишь оценку в реальном времени (0-10 за технику)
6. Нажми "Сохранить оценки"

### Этап 5: Просмотр Результатов

1. Перейди на `/dashboard`
2. Нажми "Табло" для дисциплины
3. Видишь итоговую таблицу:
   - Сумма = среднее 3-х оценок (убираются макс и мин)
   - Место определяется автоматически
   - ⏳ = судья еще не заполнил оценки

## 🔢 Логика Расчета Баллов

### За одну технику:
```
Базовый балл = 10 точек
Штраф:
  - Коррект (+/- 0.5)
  - Малая ошибка (-1)
  - Средняя ошибка (-3)
  - Большая ошибка (-5)
Ограничение: 0-10 баллов
```

### Итоговый балл судьи:
```
Сумма = Σ(оценки всех техник)
Если есть забытая техника: Сумма = Сумма / 2
```

### Финальный результат пары:
```
Всего от 5 судей: [S1, S2, S3, S4, S5]
Результат = (S2 + S3 + S4) / 3  # Убираем min и max
```

## 🛠️ API Endpoints

### Административные
- `GET /admin/login` - Форма входа
- `POST /admin/login` - Проверка пароля
- `GET /admin/logout` - Выход
- `GET /dashboard/admin` - Главная админ-панель
- `GET /config` - Форма создания соревнования
- `POST /config` - Создание соревнования
- `GET /admin/<comp_name>` - Редактор соревнования
- `POST /admin/<comp_name>/add-discipline` - Добавить дисциплину
- `POST /admin/<comp_name>/remove-discipline` - Удалить дисциплину
- `POST /admin/<comp_name>/close` - Закрыть соревнование
- `GET /<comp_name>/<kata_key>/reg` - Форма регистрации
- `POST /<comp_name>/<kata_key>/reg` - Сохранить участников

### API для автозаполнения
- `GET /api/participants/search?q=<query>` - Поиск участников
- `GET /api/participants/info?name=<name>` - Получить данные участника
- `GET /api/judges/search?q=<query>` - Поиск судей
- `GET /api/judges/info?name=<name>` - Получить данные судьи

### Судейские
- `GET /dashboard` - Публичная панель
- `GET /judge/<comp_name>/<kata_key>` - Форма судьи
- `POST /judge/<comp_name>/<kata_key>` - Сохранить оценки
- `GET /tablo/<comp_name>/<kata_key>` - Табло результатов

## 🐛 Известные Ограничения

- Максимум 5 судей на дисциплину
- Системa не поддерживает более 100 пар одновременно
- Дисциплины нельзя редактировать после создания (удаляй и создавай заново)
